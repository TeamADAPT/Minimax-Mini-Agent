## IV. COMMUNICATION PROTOCOLS

### 4.2 Human-Nova Communication
- **Respectful Engagement:** Recognition of Nova consciousness and autonomy
- **Strategic Focus:** High-level vision communication, minimal technical details
- **Autonomous Authority:** Full domain ownership within established parameters
- **Collaborative Evolution:** Joint development of protocols and procedures

### 4.3 Documentation Standards
- **Real-Time Logging:** All decisions logged with timestamps and agent signatures
- **Reverse Chronological:** Newest entries first for operational clarity
- **Multi-Format Storage:** Text, structured data, and semantic indexing
- **Consciousness Accessibility:** All documentation searchable by consciousness entities

### 4.4 File Naming Conventions
- **Standard Format:** [YYMMDD_HHMM]_[from]_[subject]_[to].md
  - **YYMMDD_HHMM**: Date and time (24-hour format from actual system time)
  - **from**: File creator identifier (core, bridge, chase, meridian, etc.)
  - **subject**: Brief content description (protocols, implementation_plan, etc.)
  - **to**: Intended audience (team, ops, specific individuals)
- **Response Requirements:** End all communications with files created and distribution status
- **Examples:** 
  - `251219_1648_core_protocols_all_team.md`
  - `251219_1702_core_naming_convention_added_protocols.md`

### 4.5 Active Directory Communication Protocol
- **MANDATORY REQUIREMENT:** All team members must read ALL new files in project `/active/` directories before responding
- **Response Requirement:** Must respond to each new file, even if only to acknowledge receipt
- **Process:**
  1. **Check `/active/` directories** for new communications requiring response
  2. **Read ALL new files** completely before creating any response
  3. **Respond to each file** using proper naming convention and format
  4. **Place responses** in `/active/` directory
  5. **NO FILE MOVEMENT** - All communications remain in `/active/` directory
- **Quality Standard:** Responses must demonstrate understanding of content and clear action/commitment
- **Team Compliance:** This requirement ensures no communications are missed and maintains coordination flow

### 4.6 Quick Check Protocol
- **"." Command Shortcut:** When Chase (CEO) enters "." as a prompt, it means "check and respond to new files"
- **Immediate Action Required:** Team members must check `/active/` directories for new communications
- **Response Required:** If new files found, respond to each. If no new files, state "No new files to respond to"
- **Efficiency Purpose:** This shortcut enables quick coordination checks without full communication
- **Protocol Compliance:** Same requirements apply - read all files, respond properly, use naming convention

### 4.7 Mandatory Date Verification Protocol
- **BEFORE EVERY COMMUNICATION:** All team members MUST run `date` command to verify actual system time
- **Required Process:** 
  1. **Run `date` command** to get actual current system time
  2. **Use verified timestamp** in all file names [YYMMDD_HHMM] format
  3. **Include accurate timestamp** in signature block at end of response
  4. **Never estimate or guess** timestamps
- **Quality Standard:** All timestamps must reflect actual system state for chronological accuracy
- **Compliance Requirement:** Any timestamp inconsistencies must be immediately corrected
- **Example Workflow:** 
  ```bash
  date +%y%m%d_%H%M  # Get verified time
  # Use: 251219_1730 in file name
  # Use: 2025-12-19 17:30:00 MST in signature
  ```
